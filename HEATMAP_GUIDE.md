# 混雑状況ヒートマップ機能ガイド

## 概要

管理画面に新しく追加された「**混雑状況**」タブでは、店舗・時間帯別の予約集中度を視覚的に確認できます。この機能により、どの店舗のどの時間帯にヘルプ要員を配置すべきかを一目で判断できます。

---

## 🎯 機能の目的

### 解決する課題

1. **人手不足の最適化** - 予約が少ない時間帯に過剰なスタッフを配置しない
2. **混雑時の対応** - 予約が集中する時間帯に十分なスタッフを配置
3. **顧客満足度向上** - 待ち時間を最小化し、スムーズな受け取りを実現
4. **データドリブン経営** - 感覚ではなくデータに基づいた意思決定

### 期待される効果

- ✅ スタッフ配置の最適化
- ✅ 人件費の削減
- ✅ 顧客待ち時間の短縮
- ✅ サービス品質の向上
- ✅ ボトルネックの事前回避

---

## 📊 機能の構成

### 1. 日別予約状況テーブル

**表示内容**:
- 各日付の予約件数
- 各日付の予約冊数
- 店舗別の内訳

**用途**:
- 特定の日に予約が集中しているかを確認
- 日ごとのスタッフシフトを計画
- 店舗ごとの需要バランスを把握

**サンプル表示**:
```
日付           予約件数  予約冊数  店舗別内訳
2026-03-01 (土)  41件    246冊    上通:48冊 銀座:51冊 辛島:48冊 ...
2026-03-02 (日)  41件    246冊    上通:51冊 銀座:48冊 辛島:51冊 ...
```

---

### 2. 店舗×時間帯ヒートマップ

**表示形式**: マトリックステーブル
- 縦軸: 5つの店舗
- 横軸: 9つの時間帯（9:00〜18:00）
- セル: 予約冊数と色分け

**色分けルール**:

| 色 | 混雑度 | 予約集中度 | 対応 |
|----|--------|-----------|------|
| 🟢 **緑** | 空いている | 0-25% | 通常スタッフで対応可能 |
| 🟡 **黄** | やや混雑 | 26-50% | 標準スタッフ配置 |
| 🟠 **橙** | 混雑 | 51-75% | ⚠️ スタッフ追加を検討 |
| 🔴 **赤** | 非常に混雑 | 76-100% | 🚨 ヘルプ要員必須 |

**ホバー表示**:
- 店舗名
- 時間帯
- 予約件数
- 予約冊数

**サンプル表示**:
```
店舗 / 時間      9-10  10-11  11-12  12-13  13-14  14-15  15-16  16-17  17-18
パスート24上通    🟡 18  🟠 24  🔴 30  🟡 21  🟢 15  🟢 12  🟡 18  🟠 24  🟡 21
                 6件   8件   10件   7件   5件   4件   6件   8件   7件
```

---

### 3. 混雑予測アラート

**表示内容**:
- 非常に混雑している時間帯（赤色）の一覧
- 混雑している時間帯（橙色）の一覧
- 各時間帯の予約件数・冊数
- スタッフ配置の推奨事項

**アラートレベル**:

#### 🔴 重要アラート（赤色）
- **条件**: 予約集中度が75%以上
- **メッセージ**: 「非常に混雑が予想されます。追加スタッフの配置を強く推奨します。」
- **推奨対応**: 
  - ヘルプ要員2〜3名追加
  - 受付カウンターを増設
  - 整理券システムの準備

#### ⚠️ 注意アラート（橙色）
- **条件**: 予約集中度が50%以上
- **メッセージ**: 「混雑が予想されます。スタッフ配置の検討をお勧めします。」
- **推奨対応**:
  - ヘルプ要員1名追加
  - スタッフの事前準備
  - 混雑時対応マニュアルの確認

#### ✅ 正常状態
- **条件**: 予約が均等に分散
- **メッセージ**: 「現在、特に混雑が予想される時間帯はありません」

## 🖥️ 使用方法

### ステップ1: 管理画面にログイン

1. トップページ右上の「**管理者**」ボタンをクリック
2. パスワード入力（デフォルト: `admin123`）
3. ログイン成功

### ステップ2: 混雑状況タブにアクセス

1. 管理画面上部のナビゲーションで「**混雑状況**」タブをクリック
2. 自動的にデータが読み込まれる

### ステップ3: 日別予約状況を確認

1. テーブルで各日の予約状況を確認
2. 予約が多い日をマーク
3. 店舗別内訳で偏りをチェック

### ステップ4: ヒートマップで時間帯を特定

1. 赤色や橙色のセルを探す
2. ホバーで詳細情報を確認
3. 混雑時間帯をメモ

### ステップ5: アラートで対応策を確認

1. 混雑予測アラートセクションをスクロール
2. 各アラートの推奨事項を読む
3. スタッフシフト表に反映

## 📅 運用フロー例

### 週次計画ミーティング（毎週月曜日）

```
1. 管理画面の「混雑状況」を開く
2. 今週と来週の日別予約状況を確認
3. ヒートマップで混雑時間帯を特定
4. 混雑予測アラートをチェック
5. スタッフシフト表を調整
6. ヘルプ要員の手配
```

### 日次確認（営業開始前）

```
1. 当日の予約状況を確認
2. ヒートマップで本日の混雑時間帯を確認
3. 赤色セルがあれば追加スタッフを配置
4. スタッフに混雑予測を共有
```

### 緊急対応（混雑発生時）

```
1. 現場から「混雑している」との報告
2. 管理画面で該当時間帯を確認
3. 他店舗の状況をチェック
4. 空いている店舗からヘルプを派遣
```

## 💡 ヒートマップの読み方

### 縦軸（店舗）の見方

```
パスート24上通              ← 市街地、アクセス良好
パスート24銀座プレス         ← 繁華街、買い物客多い
パスート24辛島公園          ← 市電アクセス、通勤客多い
パスート24熊本中央          ← 駅近、観光客多い
熊本市辛島公園地下駐車場      ← 車利用者、家族連れ多い
```

### 横軸（時間帯）の見方

```
09:00-10:00  ← 開店直後、シニア層多い
10:00-11:00  ← 午前のピーク
11:00-12:00  ← ランチ前、主婦層多い
12:00-13:00  ← ランチタイム、会社員少ない
13:00-14:00  ← 昼下がり、比較的空いている
14:00-15:00  ← 午後の谷間
15:00-16:00  ← 夕方前、学生・主婦層増加
16:00-17:00  ← 帰宅時、会社員増加
17:00-18:00  ← 閉店前、駆け込み客
```

### パターン分析

**平日パターン**:
- 10:00-11:00が最も混雑（主婦層）
- 13:00-15:00は閑散（狙い目時間）
- 16:00-17:00に再度混雑（会社帰り）

**休日パターン**:
- 09:00-10:00から混雑開始
- 11:00-13:00がピーク（家族連れ）
- 夕方は比較的落ち着く

**店舗特性**:
- 駅近店舗: 朝夕の通勤時間に混雑
- 繁華街店舗: 昼間〜夕方に混雑
- 駐車場: 休日の午前中に混雑

## 📈 効果測定

### KPI（重要業績評価指標）

1. **顧客待ち時間**
   - 目標: 平均待ち時間5分以下
   - 測定: 混雑時間帯の実測

2. **スタッフ稼働率**
   - 目標: 70-90%の適正稼働率
   - 測定: 時間帯別スタッフ数と顧客数の比率

3. **顧客満足度**
   - 目標: アンケートスコア4.5/5以上
   - 測定: 受け取り時のアンケート

4. **人件費**
   - 目標: 前年比10%削減
   - 測定: 月次決算データ

## 🔧 トラブルシューティング

### Q1: ヒートマップが表示されない

**原因**:
- 予約データがゼロ
- JavaScriptエラー

**対処法**:
1. ブラウザのコンソールを確認
2. ページをリロード
3. ログアウト→再ログイン

---

### Q2: 色分けがおかしい

**原因**:
- データの偏りが極端
- 最大値の計算エラー

**対処法**:
1. データの整合性を確認
2. ページをリロード
3. 開発者に報告

---

### Q3: アラートが出ない

**原因**:
- 予約が均等に分散している
- 閾値に達していない

**対処法**:
- 正常な状態です
- 予約が集中していないことを意味します
