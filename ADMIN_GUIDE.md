# 管理者認証システム - 使用ガイド

## 🔐 管理画面へのアクセス方法

### 方法1: トップページから（推奨）

1. **トップページにアクセス**
   - URL: `https://your-domain.pages.dev/`

2. **管理者ボタンをクリック**
   - ページ右上にある「**管理者**」ボタン（グレー背景、盾アイコン付き）をクリック

3. **ログイン画面で認証**
   - パスワード入力欄にパスワードを入力
   - デフォルトパスワード: `admin123`
   - 「ログイン」ボタンをクリック

4. **管理画面へ自動遷移**
   - 認証成功後、自動的に管理画面が表示されます

### 方法2: 直接URLアクセス

1. **管理画面URLにアクセス**
   - URL: `https://your-domain.pages.dev/admin`

2. **未認証の場合**
   - 「アクセスが拒否されました」画面が表示されます
   - 「トップページへ戻る」ボタンで戻り、方法1でログイン

3. **認証済みの場合**
   - そのまま管理画面が表示されます

---

## 🔑 認証の仕組み

### トークンベース認証
- ログイン成功時、セキュアなトークンが発行されます
- トークンは `localStorage` に保存されます
- トークンの有効期限: **24時間**

### 自動認証チェック
- 管理画面アクセス時、自動的にトークンの有効性を確認
- 有効期限切れの場合、自動的にログイン画面へリダイレクト

---

## 🖥️ 管理画面の機能

### ヘッダー
- **タイトル**: プレミアム商品券 管理画面
- **ログアウトボタン**: 右上の赤いボタンでログアウト

### ナビゲーションタブ
1. **ダッシュボード** - 全体統計とグラフ
2. **予約一覧** - 予約管理とフィルター検索
3. **予約検索** - 個別予約の検索

---

## 🔓 ログアウト方法

1. **ログアウトボタンをクリック**
   - 管理画面右上の「**ログアウト**」ボタンをクリック

2. **確認ダイアログ**
   - 「ログアウトしますか？」という確認が表示されます
   - 「OK」でログアウト、「キャンセル」で継続

3. **トップページへリダイレクト**
   - ログアウト後、自動的にトップページへ戻ります
   - トークンは削除されます

## 🔐 セキュリティのベストプラクティス

### 本番環境での設定

**1. パスワードを必ず変更する**

開発環境（.dev.vars）:
```bash
# .dev.vars ファイルを編集
ADMIN_PASSWORD=your_strong_password_here
```

本番環境（Cloudflare Pages）:
```bash
# Wranglerで環境変数を設定
npx wrangler pages secret put ADMIN_PASSWORD --project-name webapp
```

**2. 強固なパスワードを使用**
- 最低12文字以上
- 大文字・小文字・数字・記号を組み合わせる
- 推測されやすいパスワード（admin, password等）は避ける

例: `Pr3m1um!V0uch3r@2026`

**3. 定期的なパスワード変更**
- 3ヶ月ごとにパスワードを変更
- パスワード変更後は全管理者に通知

**4. トークンの有効期限管理**
- 現在の有効期限: 24時間
- 必要に応じて短縮可能（コード変更が必要）

## ⚠️ トラブルシューティング

### ログインできない場合

**症状**: パスワードエラーが表示される

**解決方法**:
1. パスワードを再確認（大文字・小文字を区別）
2. デフォルトパスワード `admin123` を試す
3. `.dev.vars` ファイルの設定を確認

### 管理画面が表示されない場合

**症状**: 「アクセスが拒否されました」画面が表示される

**解決方法**:
1. トップページの「管理者」ボタンからログイン
2. ブラウザのlocalStorageをクリア
3. 再度ログイン

### トークンが期限切れの場合

**症状**: 突然ログイン画面に戻される

**解決方法**:
- 再度ログインする（24時間以上経過している場合は正常な動作）

## 🎯 管理者の操作フロー例

### 日次業務フロー

**朝の確認**:
1. トップページの「管理者」ボタンからログイン
2. ダッシュボードで予約状況確認
3. 当日の予約を確認（予約一覧で日付フィルター）

**予約受付時**:
1. 予約検索タブで予約ID検索
2. 予約内容を確認
3. 商品券を手渡し
4. ステータスを「受取完了」に変更

**終業時**:
1. 本日の受取状況を確認
2. CSV出力で記録を保存
3. ログアウト
