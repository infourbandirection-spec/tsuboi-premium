# デモンストレーション用データセットアップガイド

## 概要

デモンストレーション用に在庫を **990冊** に設定し、残り **10冊** の状態にします。

これにより、予約フォームでの動的な冊数制限や、在庫少量時の動作を実演できます。

---

## 🎯 デモ用設定の目的

### なぜ990冊なのか？

- **残り10冊** の状態で、予約システムの動作を確認しやすい
- **在庫少量警告** の表示テストが可能
- **動的冊数制限** の動作確認（例: 残り3冊 → 1〜3冊のみ選択可能）
- **在庫上限到達** のシミュレーションが容易

---

## 📝 reset_to_990.sql の内容

このSQLファイルは以下の処理を行います:

```sql
-- 1. 既存の予約データをすべて削除
DELETE FROM reservations;

-- 2. 165件 × 6冊 = 990冊のデモデータを挿入
INSERT INTO reservations (
  reservation_id, birth_date, full_name, phone_number, 
  quantity, store_location, pickup_date, pickup_time_slot, 
  status, created_at
)
VALUES 
  ('PRE-20260225-DEMO001', '1980-01-02', 'デモユーザー1', '090-0001-0001', 6, 'パスート24上通', '2026-03-01', '10:00-11:00', 'reserved', datetime('now')),
  ('PRE-20260225-DEMO002', '1980-01-03', 'デモユーザー2', '090-0001-0002', 6, 'パスート24銀座プレス', '2026-03-01', '11:00-12:00', 'reserved', datetime('now')),
  ...
  ('PRE-20260225-DEMO165', '1980-01-26', 'デモユーザー165', '090-0001-0165', 6, '熊本市辛島公園地下駐車場', '2026-03-02', '12:00-13:00', 'reserved', datetime('now'));
```

**データ特徴**:
- 予約ID: `PRE-20260225-DEMO001` 〜 `PRE-20260225-DEMO165`
- 電話番号: `090-0001-0001` 〜 `090-0001-0165`
- 全て6冊予約（最大値）
- 5店舗に均等分散
- ステータス: すべて `reserved`（予約済み）

## 🎬 デモンストレーションのシナリオ例

### シナリオ1: 動的冊数制限のデモ

**現在の状態**: 残り10冊

1. **予約フォームにアクセス**
2. **ステップ4（冊数選択）** で、選択可能な最大値が **6冊** であることを確認
3. **新規予約（5冊）を実行** → 成功、残り5冊に
4. **再度予約フォームにアクセス**
5. **ステップ4** で、選択可能な最大値が **5冊に減少** していることを確認 ✅

### シナリオ2: 在庫少量警告のデモ

**現在の状態**: 残り10冊

1. 予約フォームで **6冊を予約** → 残り4冊
2. 再度フォームにアクセス
3. **「残り冊数が少なくなっています（残り 4 冊）」** という警告が表示される ✅
4. 選択可能な冊数が **1〜4冊** に制限される ✅

### シナリオ3: 在庫上限到達のデモ

**現在の状態**: 残り10冊

1. **10冊分の予約を実行** （複数回に分けて）
2. 在庫が **0冊** になる
3. **トップページにアクセス** → 「予約受付終了」が表示される ✅
4. `isAccepting: false` になり、新規予約フォームが非表示になる ✅

### シナリオ4: 競合状態のデモ

**現在の状態**: 残り10冊

1. **テストツールにアクセス**
   https://3000-ias0xb1bnq0w0e36xso19-cc2fbc16.sandbox.novita.ai/static/test-race-condition.html
2. **在庫を2冊まで減らす** （8冊分の予約を実行）
3. **「3人同時予約テストを実行」** ボタンをクリック
4. **結果確認**:
