# プレミアム商品券予約システム

熊本市のプレミアム商品券の予約から受け取りまでを効率的に管理する包括的なウェブアプリケーションです。

## ✨ 主な機能

### システム運用フロー

#### **Phase 1: 応募・抽選期間（～3月10日 23:59）**
1. **応募受付開始**: 予約フォームから応募を受け付け
   - 上限なく応募を受け付ける（1200冊以上も可能）
   - すべての応募は `lottery_status='pending'`
   - 混雑状況には全応募者が表示される

2. **応募締切**: 2026年3月10日 23:59
   - 管理画面で手動で予約受付を停止
   - 応募者は予約照会ページで応募状況を確認可能

3. **抽選実行**: 2026年3月11日
   - 管理画面の「抽選」タブで抽選を実行
   - 1000冊に収まるように自動調整
   - 当選者: `lottery_status='won'`
   - 落選者: `lottery_status='lost'`
   - 混雑状況が当選者のみに切り替わる

4. **抽選結果確認**: 
   - 当選者掲示板ページ（/lottery-results）で公開
   - 予約照会ページで個別確認可能

#### **Phase 2: 受け取り期間（3月16日～18日）**
- 当選者が指定日時に店舗で受け取り
- スタッフが管理画面で受取完了を登録
- 混雑状況は当選者のみ表示される

#### **Phase 3: 追加予約期間（3月19日以降）**
1. **予約受付再開**: 管理画面で手動で再開
   - Phase 2モードに切り替え
   - 日付を自由にカレンダーで選択可能
   - 1000冊上限で即時予約確定（抽選なし）
   - `lottery_status='n/a'`（抽選対象外）

2. **在庫管理**:
   - リアルタイムで残数をチェック
   - 1000冊到達で自動停止
   - 手動でも停止可能

## 💾 データベース構造

### reservations テーブル（予約情報）
```sql
- id: 予約ID（自動採番）
- reservation_id: ユーザー向け予約ID（PRE-YYYYMMDD-XXXXXX）
- birth_date: 生年月日
- full_name: 氏名
- phone_number: 電話番号
- quantity: 冊数（1～6）
- store_location: 受け取り店舗
- pickup_date: 受け取り日
- pickup_time_slot: 受け取り時間帯
- status: ステータス（reserved/completed/canceled）
- created_at: 予約日時
- updated_at: 更新日時
```

### stores テーブル（店舗マスター）
```sql
- パスート24上通（24時間営業）
- パスート24銀座プレス（24時間営業）
- パスート24辛島公園（24時間営業）
- パスート24熊本中央（24時間営業）
- 熊本市辛島公園地下駐車場（7:00～22:00）
```

### system_settings テーブル（システム設定）
```sql
- max_total_books: 総発行上限冊数（1000）
- is_accepting_reservations: 予約受付中フラグ
```

## 🔒 排他制御と競合状態対応

### 実装されている二重チェック機構

本システムでは、**残り2冊で3人が同時に予約**するような競合状態でも、データ整合性を完全に保証します。

#### シナリオ例
- **初期状態**: 残り2冊
- **User A**: 2冊予約 → ✅ **成功**（予約ID発行）
- **User B**: 1冊予約 → ❌ **失敗**（「予約上限に達しました」エラー）
- **User C**: 1冊予約 → ❌ **失敗**（「予約上限に達しました」エラー）
- **結果**: 在庫1000冊（超過なし）、予約受付自動停止

#### 排他制御の仕組み

```typescript
// 【第1段階】Cloudflare D1 BATCHによる初期チェック
const results = await db.batch([
  db.prepare('SELECT SUM(quantity) as total FROM reservations WHERE status = "reserved"'),
  db.prepare('SELECT id FROM reservations WHERE phone_number = ? AND status = "reserved"').bind(phoneNumber)
])

// 在庫チェック
if (currentReserved + requestedQuantity > 1000) {
  return error('予約上限に達しました')
}

// 【第2段階】挿入直前の最終防衛ライン
const finalCheck = await db.prepare('SELECT SUM(quantity) as total FROM reservations WHERE status = "reserved"').first()

if (finalReserved + requestedQuantity > 1000) {
  return error('他の方の予約が完了し、予約上限に達しました')
}

// 問題なければ挿入実行
await db.prepare('INSERT INTO reservations ...').run()
```

#### テスト方法

1. **在庫を998冊に設定**（テスト用）
2. **3人同時予約を実行**（競合状態テストツール使用）
3. **結果確認**:
   - 1人のみ成功（2冊予約）
   - 2人は失敗（適切なエラーメッセージ表示）
   - 最終在庫: 1000冊（整合性保証）

詳細は [`RACE_CONDITION_REPORT.md`](./RACE_CONDITION_REPORT.md) を参照。
